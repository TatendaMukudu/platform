## 1. What shipped today (in order)

Today was two arcs: **(a) build the Attention Engine's proactive behaviour**, then **(b) turn the
architecture into enforced governance**. Each landed as a small, committed, independently-green step.

**Attention Engine (product)**
- **Polarity + "Your Attention".** Every insight now carries a polarity (`risk / progress / milestone /
  opportunity / neutral`) that is *independent of priority*. Home groups into **Needs attention · Worth
  celebrating · Opportunities**. Same kernel, different projections.
- **Positive momentum is first-class.** Recoveries, quiet improvements, and a deterministic **milestone**
  (check-in streak / personal best) surface as celebrations — not just problems.
- **Opportunities**, self-audience only, framed as *questions* (never verdicts or predictions).
- **The proactive opening.** The assistant now greets from the artifacts it consumes: *"Good morning,
  Mia. Here's what deserves your attention today,"* leading with a win when there is one, then needs-
  attention, then opportunity, with an invitation to explore. Deterministic, no AI key required.
- **Explore prompts** on every insight so wins and risks both start conversations ("what helped create
  this?" / "would you like to explore what changed?") — never diagnosing or assuming cause.

**Governance (platform)**
- **Attention Behaviour layer** (`ai/behaviour.js`) — extracted delivery (grouping, ordering, volume,
  opening, silence) into a *pure* module that **imports nothing**. Projection (`ai/proactive.js`) now
  owns only the artifact + audience-safety. The boundary is a *file* boundary.
- **Collapsed the parallel surfaces.** `/api/me/context` and the assistant opening now *consume* the one
  pipeline instead of recomputing proactively. Deleted the duplicate Home render (a second greeting, a
  second "noticed" list, a duplicate "prepared").
- **Reconciled the last privacy leak.** The legacy leader timeline exposed mood numbers and quoted
  check-ins; a leader viewing anyone else now gets **direction words + authorised work evidence only**.
- **Enforced the constitution in CI.** `governance-smoke.js` fails the build if proactive delivery is
  written anywhere except `ai/behaviour.js`. It already caught and forced out a residual (me/context's
  independent opening). "Architectural gravity" is now a test, not a hope.

**Responsibilities, now enforced:** kernel owns truth · projection owns visibility · behaviour owns
delivery · assistant owns conversation · privacy owns boundaries. Never overlapping.

## 2. Are we proactive yet?

**Yes — in the specific, honest sense we built, and deliberately not in one other sense.**

- ✅ **Proactively attentive.** Open IntelliQ and, before you type, it has already organised your world:
  a greeting, what's worth celebrating, what needs you, and opportunities — grounded only in canonical
  evidence, privacy-safe, and calm when there's nothing ("you're in a steady place" is a valid answer).
- ✅ **The assistant opens conversations** from verified artifacts instead of waiting for a prompt, and
  reinforces the good as readily as it surfaces the concerning.
- ✅ **It surfaces, it never acts.** Every suggestion is proposal-gated. Proactive *attentiveness*, not
  proactive *action*.
- ⛔ **Not a notifier (by design).** It is proactive *when you open it* (pull), not across channels
  (push). There is no notification/scheduling/timing engine — that was explicitly out of scope until
  there's a real delivery channel and a pilot to justify it. So: "proactive within the experience," yes;
  "it pings your phone," no, on purpose.

Net: **the defining interaction is now proactive attentiveness.** That's the thing the vision asked for.
The push/notification layer is a future, deliberate decision — not a gap we forgot.

## 4. Architecture state — the honest scorecard

| Area | State |
|---|---|
| One kernel / one detector / one confidence engine | ✅ unchanged, authoritative |
| One projection (artifact + audience-safety) | ✅ `ai/proactive.js` |
| One behaviour layer (delivery) | ✅ `ai/behaviour.js`, pure, CI-enforced |
| Every proactive surface is a consumer | ✅ Home, opening, me/context, leader support, timeline |
| Privacy identical across surfaces | ✅ incl. the reconciled leader timeline |
| Works with no AI key | ✅ deterministic throughout |
| Constitution enforced in CI | ✅ `governance-smoke.js` |

**Not yet collapsed (deliberate, documented):**
- The **leader Platform Intelligence org briefing** still computes independently. This is *organizational*
  attention — aggregates/comparisons/population stats — a genuinely different problem from subject
  attention. It should get its own design (aggregation → projection → behaviour, **no new reasoning
  engine**, minimum-cohort suppression to prevent small-N re-identification) — and that design is best
  written *after* the pilot, informed by reality, not before.
- Legacy org analytics still read the canonicalised `memberResults` mirror (documented; off the assistant
  truth path).
- "Learn from success" (recurring-pattern memory) is **deferred** — it needs retrieval/similarity we won't
  fake. Truth over sophistication.

## 6. Recommendation

The highest-leverage next move is **not more code — it's the pilot.** The architecture is now coherent and
self-enforcing; what it hasn't met is a real user. The pilot is the forcing function that will tell us

observe → org-attention design doc → v2.** Before the pilot, close the accessibility P0.
